# DeepSeek Protocol Pipeline

This folder contains a two-stage demo workflow:

1. `v4-flash` extracts verbatim preparation evidence from a paper markdown.
2. `v4-pro` converts the extracted evidence into a protocol JSON that matches the shot format.

## Files

- `deepseek_protocol_pipeline.py`: main pipeline script
- `deepseek_protocol_config.example.json`: config template with API key and model placeholders
- `prompts/stage1_extract_preparation_evidence.txt`: stage-1 prompt template
- `prompts/stage2_generate_protocol.txt`: stage-2 prompt template

## How To Use

1. Copy `deepseek_protocol_config.example.json` to `deepseek_protocol_config.json`.
2. Fill in your real API key and model names manually.
3. Run the script with only the markdown path.

Example command in PowerShell:

```powershell
python .\workflow\deepseek_protocol_pipeline.py .\markdown\1-s2.0-S0167577X2400404X-main.md
```

Default behavior:

- Config file: `.\workflow\deepseek_protocol_config.json`
- Shot protocol: `.\dataset\ODMA_GelMA_RSF_CHI_protocol.json`
- Intermediate output directory: `.\dataset\output`
- Final protocol directory: `.\dataset`
- Protocol name: generated by `stage2` from extracted evidence

## Output Files

The intermediate output directory will contain:

- `<protocol_name>_stage1_prompt.txt`: final prompt sent to `v4-flash`
- `<protocol_name>_stage1_raw_response.txt`: raw stage-1 model response
- `<protocol_name>_stage1_segments.json`: parsed stage-1 evidence segments
- `<protocol_name>_stage2_prompt.txt`: final prompt sent to `v4-pro`
- `<protocol_name>_stage2_raw_response.txt`: raw stage-2 model response

The final protocol JSON will be written to:

- `.\dataset\<protocol_name>.json`

## Notes

- The script uses only Python standard library modules.
- The script does not install dependencies or run tests.
- The shot JSON is used as a structural example, not as content to be copied.
- `Protocol` is generated by `stage2`, while intermediate artifact filenames still use the markdown stem for stability.
